Docker 学习

Docker 学习

一、了解容器及容器演进

1、什么是容器Container

可容纳资源的都是容器，背包、操作系统、Nginx；

2、容器特点

  • 使对象资源标准化
    • 测试环境部署不再混乱，要部署一套只需要新建和运行一个容器即可，无需担心运行环境可能被其他人破坏
    • 新人入职环境搭建方便简易
  • 提供统一标准化的操作方式，不关注内部细节
    • 数据库中间件MyCat实现MySQL分库分表

3、虚拟化

虚拟化是一种资源管理技术，是将计算机的各种实体资源，如服务器、网络、内存及存储等，予以抽象、转换后呈现出来，打破实体结构间的不可切割的障碍，使用户可以比原本的配置更好的方式来应用这些资源。这些资源的新虚拟部分是不受现有资源的架设方式，地域或物理配置所限制。一般所指的虚拟化资源包括计算能力和数据存储。

  • 系统虚拟化：在 Host 中通过 Hypervisor 层实现安装多个 GuestOS，每个 GuestOS 都有自己的内核，和主机的内核不同，GuestOS 之间完全隔离。比如VMWare。
  • 容器虚拟化：使用 Linux 内核中的 namespaces 和 cgroups 实现进程组之间的隔离。是用内核技术实现的隔离，所以它是一个共享内核的虚拟化技术。比如Docker。

4、容器与LXC（Linux Container）

LXC即Linux Container的缩写，是一种内核虚拟化技术，可提供轻量级的虚拟化，方便隔离进程和资源，而Docker的底层是基于Linux的内核LXC技术实现的。

如果windows操作系统上面没有该技术则无法使用Docker，Window10上面有Liniux内核，可以直接安装，但是windows7要使用Docker就要先安装Docker Toolbox来支持。

2、Docker的架构（与GIT对比）

2-1、Docker组成部分

  • 客户端

    提供操作docker服务端入口（可以命令行，也可以API方式）

  • 镜像

    提供应用镜像（应用可以是单个服务，比如mysql、nginx，也可以是一个操作系统，如centos）

    运行镜像服务，可以同一个镜像生成多个容器

  • 仓库

    提供镜像搜索（最大的docker hub，也可以搭建自己私人的）

2、镜像操作命令

2-1、*列出：docker images

  • 作用：查看所有镜像

  • 参数说明：

    • repository：镜像所在仓库名称
    • tag：镜像版本
    • image ID：镜像ID
    • created：镜像创建时间
    • size：镜像大小

2-2、搜索：docker search 名称

  • 作用：搜索镜像（从docker hub上）

  • 参数说明：

    • name：仓库名称
    • description：镜像描述
    • starts：用户评价
    • official：是否官方
    • autometed：是否自动构建，表示该镜像由dockerhub自动构建流程创建

    该命令较少使用，一般是直接在 https://hub.docker.com/ 搜索

2-3、*拉取： docker pull 镜像名称:tag

作用：从仓库拉取镜像

2-4、*删除： docker rmi 仓库名称:tag / 仓库ID

​ 注意：删除时，如镜像的image ID一致（比如将一个镜像自己打了另外一个tag，那这两个imageID就是一致的），则需要按照一定顺序进行删除，因为镜像之间有关联（reference）。

2-3、导入导出

  • 导出：
    • docker save 仓库名称:tag/imageID > /具体路径/xxx.tar.gz
    • docker save -o filename.tar 仓库名称:tag/imageID
  • 导入：
    • docker load < /具体路径/xxx.tar.gz
    • docker load -i filename.tar

3、容器操作命令

3-1、创建：docker run

  • 以交互方式运行：docker run -it –name 容器名称 仓库名称:tag/imageID /bin/bash

  • 以守护进程方式运行：docker run -d –name 容器名称 仓库名称:tag/imageID /bin/bash

    • -i : 表示运行容器
    • -t : 表示容器启动后UI进入其命令行。加入这两个参数后，容器创建就能登录进去，即分配了一个伪终端
    • –name : 为创建的容器命令
    • -v : 表示目录映射关系（前者是宿主机目录，后者是映射到宿主机上的目录），可以使用多个-v做多个目录或者文件映射。注意：最好做目录映射，在宿主机上做修改，然后共享到容器上
    • -d : 在run后面加改参数则会创建一个守护容器在后台运行（不会自动登录容器，如果只加-i -t 两个参数，创建后就会自动进入容器）
    • -p : 表示端口映射，前者是宿主机端口，后者是容器内的映射端口，可以多个-p做多个端口映射
    • -e: 表示指定环境变量

    通过run创建并进入容器之后，如果使用exit命令退出容器，则容器停止；再次进入该容器，先使用start启用容器，再使用exec、attach命令进入容器

3-2、*启动：docker start

​ docker start 容器名称或者容器ID

3-3、*进入：docker exec

​ 进入正在运行的容器命令：

  • docker exec -it 容器名称或者容器ID /bin/bash
  • docker attach 容器名称或者容器ID

两者区别：attach进入容器后如使用exit退出容器则容器会停止；exec进入容器后，使用exit退出，容器依然处于运行状态

3-4、*查看：docker ps

  • ​ docker ps：查看正在运行容器
    • docker ps -a：查看历史运行过的容器
    • docker ps -l：查看最近运行过的容器

3-5、*停止：docker stop

docker stop 容器名称或者容器ID

3-6、*删除：docker rm

docker rm 容器名称或者容器ID

3-7、*复制：docker cp

docker cp [容器名称:]源文件 [容器名称:]目标文件

docker cp /root/q.sql 容器名称:/home/backup/s.sql

注意：源文件可以是宿主机也可以是容器中文件，同样，目标文件也是一样，不过如果是容器文件则要加上容器名称

4、制作镜像命令

1、登录自己的镜像仓库账号

  • docker login [docker仓库地址] -u youraccount

    推送之前要先登录账号

2、制作镜像

  • docker commit 容器ID 仓库账号/容器名称:tag

    docker commit 65b3a0ed0e38 dockerHubAccount/testApp:2.0.0

3、推送镜像到远程

  • docker push 仓库账号/镜像名称:tag

    docker push dockerHubAccount/testApp:2.0.0

3、拉取镜像

  • docker pull https://hub.docker.com/dockerHubAccount/testApp:2.0.0

六、练习：自己搭建项目的开发环境

搭建基于Docker的随处可用、方便快捷的项目开发环境，思路：项目运行在Linux上，所以需要Centos7作为最基础的镜像，然后在Centos7的镜像上面安装PHP+MySQL+Redis+Nginx环境（或者用直接宝塔），由此得到经过处理的Centos7镜像，然后将镜像commit并push到docker hub的自己私人仓库上，并打tag。清空本地的镜像和仓库，从docker hub上面将项目pull下来并运行。
